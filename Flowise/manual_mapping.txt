=> Chat Model

    -> Purpose

        -> Configures the Chat Model used to communicate with the selected LLM.
        -> The same configuration exists in both Flowise and manual implementation.

    -> Flowise

        Chat Model Node
                ↓
        Configure

            -> API Key
            -> Model Name
            -> Temperature

    -> Spring AI

        ChatModel Bean
                ↓
        Configure

            -> API Key
            -> Model Name
            -> Temperature

    -> LangChain4j

        ChatLanguageModel
                ↓
        Configure

            -> API Key
            -> Model Name
            -> Temperature

    -> Mapping

        Flowise Chat Model Node
                    ⇔
        Spring AI ChatModel Bean
                    ⇔
        LangChain4j ChatLanguageModel

    -> Important

        -> Flowise configures the Chat Model visually.
        -> Manual implementation configures the same Chat Model in code.
        -> The Chat Model acts as the communication bridge between the application and the LLM.

=> Chat Prompt Template

    -> Purpose

        -> Creates the final prompt before sending it to the LLM.
        -> Combines the System Message and Human Message.

    -> Flowise

        Chat Prompt Template Node

    -> Spring AI

        Prompt

    -> LangChain4j

        PromptTemplate

    -> Mapping

        Flowise Chat Prompt Template
                    ⇔
        Spring AI Prompt
                    ⇔
        LangChain4j PromptTemplate

    -> Important

        -> Flowise creates the prompt visually.
        -> Manual implementation creates the same prompt in code before invoking the Chat Model.

=> LLM Chain

    -> Purpose

        -> Orchestrates the interaction between the Prompt and the Chat Model.
        -> Invokes the selected LLM and returns the generated response.

    -> Flowise

            Chat Prompt Template
                        ↓
                    LLM Chain
                      ↑
                      │
                Chat Model

    -> Spring AI

            Prompt
                ↓
            ChatClient
                ↓
            ChatModel
                ↓
            call()
                ↓
            Response

    -> LangChain4j

            Prompt
                ↓
        ChatLanguageModel
                ↓
        AiServices / LLM Chain
                ↓
            Response

    -> Mapping

        Flowise Chat Prompt Template
                    ⇔
        Spring AI Prompt
                    ⇔
        LangChain4j PromptTemplate

        Flowise Chat Model
                    ⇔
        Spring AI ChatModel Bean
                    ⇔
        LangChain4j ChatLanguageModel

        Flowise LLM Chain
                    ⇔
        Spring AI ChatClient
                    ⇔
        LangChain4j AiServices / LLM Chain

    -> Important

        -> Flowise exposes the orchestration as an LLM Chain node.
        -> In Spring AI, the orchestration is handled internally by the ChatClient.
        -> In LangChain4j, the orchestration is handled by AiServices or an LLM Chain.
        -> The execution flow remains the same; only the implementation differs.

=> Buffer Memory

    -> Purpose

        -> Stores the conversation history between the user and the AI.
        -> Enables the AI to remember previous interactions.
        -> Provides conversation history to the Conversation Chain.

    -> Flowise

            User Message
                    ↓
            Buffer Memory
                    ↓
        Conversation History

    -> Spring AI

            ChatMemory

    -> LangChain4j

            ChatMemory

    -> Mapping

        Flowise Buffer Memory
                    ⇔
        Spring AI ChatMemory
                    ⇔
        LangChain4j ChatMemory

    -> Important

        -> Buffer Memory only stores and retrieves conversation history.
        -> It does not communicate directly with the Chat Model.
        -> It is used by Conversation Chain to generate context-aware responses.

=> Conversation Chain

    -> Purpose

        -> Creates a stateful AI chatbot.
        -> Retrieves conversation history from Buffer Memory.
        -> Internally builds the final prompt.
        -> Invokes the Chat Model.
        -> Updates Buffer Memory after generating the response.

    -> Flowise

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
              Chat Model

    -> Spring AI

            ChatMemory
                  │
                  ▼
            PromptTemplate
                  ↓
              ChatClient
                  ↓
              ChatModel
                  ↓
              Response

    -> LangChain4j

            ChatMemory
                  │
                  ▼
            PromptTemplate
                  ↓
            AiServices
                  ↓
        ChatLanguageModel
                  ↓
              Response

    -> Internal Working

            User Message
                    +
            System Prompt
                    +
            Conversation History
                    ↓
            Internal Chat Prompt Template
                    ↓
                Chat Model
                    ↓
                LLM Response
                    ↓
            Update Buffer Memory

    -> Mapping

        Flowise Conversation Chain
                    ⇔
        Spring AI
            ChatClient
                +
            PromptTemplate
                +
            ChatMemory

                    ⇔
        LangChain4j
            AiServices
                +
            PromptTemplate
                +
            ChatMemory

    -> Important

        -> Conversation Chain internally creates the Chat Prompt Template.
        -> Unlike LLM Chain, it supports Memory.
        -> It hides the complete orchestration from the developer.
        -> If we implement it manually, we need to explicitly manage PromptTemplate, ChatMemory and ChatClient.
