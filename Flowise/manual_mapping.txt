=> Chat Model

    -> Purpose

        -> Configures the Chat Model used to communicate with the selected LLM.
        -> The same configuration exists in both Flowise and manual implementation.

    -> Flowise

        Chat Model Node
                ↓
        Configure

            -> API Key
            -> Model Name
            -> Temperature

    -> Spring AI

        ChatModel Bean
                ↓
        Configure

            -> API Key
            -> Model Name
            -> Temperature

    -> LangChain4j

        ChatLanguageModel
                ↓
        Configure

            -> API Key
            -> Model Name
            -> Temperature

    -> Mapping

        Flowise Chat Model Node
                    ⇔
        Spring AI ChatModel Bean
                    ⇔
        LangChain4j ChatLanguageModel

    -> Important

        -> Flowise configures the Chat Model visually.
        -> Manual implementation configures the same Chat Model in code.
        -> The Chat Model acts as the communication bridge between the application and the LLM.

=> Chat Prompt Template

    -> Purpose

        -> Creates the final prompt before sending it to the LLM.
        -> Combines the System Message, Human Message, and Prompt Variables.
        -> Supports reusable and dynamic prompts.

    -> Flowise

        Chat Prompt Template Node

        -> Supports Prompt Variables using {}.
        -> Uses Format Prompt Values to assign values to Prompt Variables.
        -> Automatically replaces Prompt Variables before sending the final prompt to the LLM.

    -> Spring AI

        PromptTemplate

        -> Defines a reusable prompt.
        -> Uses placeholders like {topic}.
        -> Variables are supplied using a Map before creating the final Prompt.

    -> LangChain4j

        PromptTemplate

        -> Defines a reusable prompt.
        -> Uses placeholders like {{topic}}.
        -> Variables are supplied before generating the final prompt.

    -> Mapping

        Flowise Chat Prompt Template
                    ⇔
        Spring AI PromptTemplate
                    ⇔
        LangChain4j PromptTemplate

    -> Example

        Prompt Template

            Explain {topic}

        Variable Mapping

            topic = Kafka

        Final Prompt

            Explain Kafka

    -> Important

        -> Flowise creates the prompt visually.
        -> Spring AI and LangChain4j create the same prompt programmatically.
        -> Prompt Variables make prompts reusable.
        -> Flowise automatically replaces Prompt Variables using Format Prompt Values.
        -> Spring AI and LangChain4j require developers to explicitly provide variable values before invoking the Chat Model.

=> LLM Chain

    -> Purpose

        -> Orchestrates the interaction between the Prompt and the Chat Model.
        -> Invokes the selected LLM and returns the generated response.

    -> Flowise

            Chat Prompt Template
                        ↓
                    LLM Chain
                      ↑
                      │
                Chat Model

    -> Spring AI

            Prompt
                ↓
            ChatClient
                ↓
            ChatModel
                ↓
            call()
                ↓
            Response

    -> LangChain4j

            Prompt
                ↓
        ChatLanguageModel
                ↓
        AiServices / LLM Chain
                ↓
            Response

    -> Mapping

        Flowise Chat Prompt Template
                    ⇔
        Spring AI Prompt
                    ⇔
        LangChain4j PromptTemplate

        Flowise Chat Model
                    ⇔
        Spring AI ChatModel Bean
                    ⇔
        LangChain4j ChatLanguageModel

        Flowise LLM Chain
                    ⇔
        Spring AI ChatClient
                    ⇔
        LangChain4j AiServices / LLM Chain

    -> Important

        -> Flowise exposes the orchestration as an LLM Chain node.
        -> In Spring AI, the orchestration is handled internally by the ChatClient.
        -> In LangChain4j, the orchestration is handled by AiServices or an LLM Chain.
        -> The execution flow remains the same; only the implementation differs.

=> Buffer Memory

    -> Purpose

        -> Stores the conversation history between the user and the AI.
        -> Enables the AI to remember previous interactions.
        -> Provides conversation history to the Conversation Chain.

    -> Flowise

            User Message
                    ↓
            Buffer Memory
                    ↓
        Conversation History

    -> Spring AI

            ChatMemory

    -> LangChain4j

            ChatMemory

    -> Mapping

        Flowise Buffer Memory
                    ⇔
        Spring AI ChatMemory
                    ⇔
        LangChain4j ChatMemory

    -> Important

        -> Buffer Memory only stores and retrieves conversation history.
        -> It does not communicate directly with the Chat Model.
        -> It is used by Conversation Chain to generate context-aware responses.

=> Conversation Chain

    -> Purpose

        -> Creates a stateful AI chatbot.
        -> Retrieves conversation history from Buffer Memory.
        -> Internally builds the final prompt.
        -> Invokes the Chat Model.
        -> Updates Buffer Memory after generating the response.

    -> Flowise

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
              Chat Model

    -> Spring AI

            ChatMemory
                  │
                  ▼
            PromptTemplate
                  ↓
              ChatClient
                  ↓
              ChatModel
                  ↓
              Response

    -> LangChain4j

            ChatMemory
                  │
                  ▼
            PromptTemplate
                  ↓
            AiServices
                  ↓
        ChatLanguageModel
                  ↓
              Response

    -> Internal Working

            User Message
                    +
            System Prompt
                    +
            Conversation History
                    ↓
            Internal Chat Prompt Template
                    ↓
                Chat Model
                    ↓
                LLM Response
                    ↓
            Update Buffer Memory

    -> Mapping

        Flowise Conversation Chain
                    ⇔
        Spring AI
            ChatClient
                +
            PromptTemplate
                +
            ChatMemory

                    ⇔
        LangChain4j
            AiServices
                +
            PromptTemplate
                +
            ChatMemory

    -> Important

        -> Conversation Chain internally creates the Chat Prompt Template.
        -> Unlike LLM Chain, it supports Memory.
        -> It hides the complete orchestration from the developer.
        -> If we implement it manually, we need to explicitly manage PromptTemplate, ChatMemory and ChatClient.

=> Input Moderation

    -> Purpose

        Validates the user's input before sending it to the LLM.

        Prevents harmful, restricted, or unwanted prompts from reaching the AI model.

    -> Flowise

        Simple Prompt Moderation

            -> Uses a configurable Deny List.
            -> Performs rule-based text matching.
            -> Blocks the prompt before invoking the LLM.
            -> Returns a custom Error Message.

        OpenAI Moderation

            -> Uses OpenAI's Moderation API.
            -> Performs AI-powered semantic moderation.
            -> Blocks unsafe prompts before invoking the LLM.

    -> Spring AI

        No dedicated Input Moderation component.

        Manual implementation is required.

        Common approaches:

            -> Spring Filter
            -> HandlerInterceptor
            -> Controller Validation
            -> Service Layer Validation
            -> OpenAI Moderation API integration

    -> LangChain4j

        No dedicated Input Moderation component.

        Manual implementation is required.

        Common approaches:

            -> Validate user input before calling ChatModel.generate()

            -> OpenAI Moderation API

            -> Custom validation logic

    -> Mapping

        Flowise Simple Prompt Moderation
                    ⇔
        Spring Boot Filter / Interceptor / Validation
                    ⇔
        LangChain4j Manual Validation

        ------------------------------------------------

        Flowise OpenAI Moderation
                    ⇔
        Spring AI OpenAI Moderation API
                    ⇔
        LangChain4j OpenAI Moderation API

    -> Internal Working

            User Prompt
                    ↓
            Input Validation
                    ↓
            Prompt Allowed?

                No
                    ↓
                Return Error

                Yes
                    ↓
                Prompt Template
                    ↓
                Chat Model
                    ↓
                LLM

    -> Example

        Deny List

            hack
            passwords
            credit card

        User Prompt

            How to hack a website?

        Result

            Prompt blocked before reaching the LLM.

    -> Important

        -> Flowise provides Input Moderation as a ready-to-use node.

        -> Spring AI and LangChain4j require developers to implement the validation logic manually.

        -> Simple Prompt Moderation is rule-based.

        -> OpenAI Moderation is AI-powered and understands the meaning of the prompt.

        -> Input Moderation should always execute before Prompt Template creation and LLM invocation.

        -> In production systems, Input Moderation is commonly implemented as middleware or request validation to ensure only safe prompts are processed.
