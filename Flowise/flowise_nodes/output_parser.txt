=> Output Parsers

    -> Purpose

        Output Parsers are used to control the format of the response generated by the LLM.
        Many people think the parser converts the LLM's response into JSON. ❌ That's not what happens.
        Instead of returning free-form text, they instruct the LLM to generate data in a specific structure such as JSON, CSV, or List.
        This makes the output predictable and easier to consume by applications.
        The parser mainly defines the expected format and validates that the response follows it.

    -> Types

        1. Structured Output Parser

            Purpose

                Forces the LLM to return data in a predefined JSON structure.

            Example

                Normal Response

                    John is 25 years old and works as a Java Developer.

                Structured Response

                    {
                        "name": "John",
                        "age": 25,
                        "profession": "Java Developer"
                    }

            Use Cases

                - User Profile Extraction
                - Resume Parsing
                - Invoice Parsing
                - API Responses
                - Backend Microservices

            Important

                This is the most commonly used Output Parser.

        2. Advanced Structured Output Parser

            Purpose

                Forces the LLM to return data that strictly follows a predefined schema (Zod Schema).

            Example

                Expected Schema

                    name : string
                    age : number
                    email : string
                    salary : number

                Invalid Response

                    {
                        "name": "John",
                        "age": "Twenty Five"
                    }

                Reason

                    age must be a number, not a string.

            Use Cases

                - Enterprise Applications
                - Banking Systems
                - Financial Applications
                - Database Insertion
                - Strict API Contracts

            Important

                Similar to validation annotations in Java.

                    @NotNull
                    @Email
                    @Min(18)

        3. CSV Output Parser

            Purpose

                Forces the LLM to return the response in CSV format.

            Example

                Product,Price
                Apple,100
                Orange,80

            Use Cases

                - Excel Reports
                - Data Analysis
                - BI Tools
                - Database Import

        4. Custom List Output Parser

            Purpose

                Forces the LLM to return only a list of values.

            Example

                [
                    "Java",
                    "Python",
                    "Go",
                    "Rust"
                ]

            Use Cases

                - Skill Lists
                - Keywords
                - Search Suggestions
                - Recommendations

    -> Comparison

        Structured Output Parser

            Output : JSON
            Validation : Basic
            Best For : General AI Applications

        Advanced Structured Output Parser

            Output : JSON
            Validation : Strict Schema Validation
            Best For : Production & Enterprise Applications

        CSV Output Parser

            Output : CSV
            Validation : No
            Best For : Reports and Analytics

        Custom List Output Parser

            Output : List / Array
            Validation : No
            Best For : Keywords, Tags and Recommendations

    -> Learning Order

        1. Structured Output Parser
        2. Advanced Structured Output Parser
        3. Custom List Output Parser
        4. CSV Output Parser
