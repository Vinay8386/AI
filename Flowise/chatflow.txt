====================================! Task 1: Create our first Chatflow !=======================

    Objective:
        -> Build a basic AI application by connecting a Chat Prompt Template, Chat Model, and LLM Chain.
        -> Understand how Flowise visually represents the components used in manual AI application development.

    -> Definition

        -> Chatflow is a visual workflow used to build AI applications in Flowise.
        -> It is built on top of LangChain.
        -> Instead of writing code, we connect different LangChain Components (Nodes) visually.
        -> Each Node performs a specific responsibility, and together they form a complete AI application.

    -> Architecture

            User
                ↓
        Chat Prompt Template
                ↓
            LLM Chain
              ↑
              │
          Chat Model
                ↓
            LLM Response

    -> Components

        -> Chat Prompt Template
            -> Creates the final prompt by combining the System Message and Human Message.

        -> Chat Model
            -> Acts as the communication bridge between the application and the selected LLM.

        -> LLM Chain
            -> Receives the Prompt and Chat Model.
            -> Invokes the selected LLM.
            -> Returns the generated response.

    -> Execution Flow

            User enters a question
                    ↓
            Chat Prompt Template combines:
                -> System Message
                -> Human Message
                    ↓
            Final Prompt is created
                    ↓
            LLM Chain receives:
                -> Final Prompt
                -> Chat Model
                    ↓
            Chat Model sends the request to the configured LLM
                    ↓
            LLM generates the response
                    ↓
            LLM Chain returns the response
                    ↓
            Response is displayed to the user

    -> Current Chatflow

            Chat Prompt Template
                        ↓
                    LLM Chain
                      ↑
                      │
                Google Gemini

    -> Use Cases

        -> AI Chatbot
        -> Question Answering
        -> RAG Applications
        -> PDF Chat
        -> Knowledge Assistant
        -> Customer Support Bot

    -> Limitations

        -> Best suited for single-agent applications.
        -> Limited support for complex branching and multi-agent workflows.
        -> For advanced AI applications, Agentflow is recommended.

    -> Important

        -> Chatflow is built on top of LangChain.
        -> Every Node represents a LangChain Component.
        -> We build an AI application by connecting the required Nodes together.
        -> The Chat Prompt Template prepares the prompt.
        -> The Chat Model communicates with the selected LLM.
        -> The LLM Chain orchestrates the complete execution flow.

====================================! Task 2: Add Memory to Chatflow !=======================

    Objective:

        -> Enable the Chatflow to remember previous conversations.
        -> Understand how conversation history is maintained and how it improves contextual responses.

    -> Why Conversation Chain?

        -> In Task 1, we used LLM Chain to execute a single prompt.
        -> LLM Chain is a stateless chain, meaning it does not remember previous conversations.
        -> Therefore, it cannot be directly used with Buffer Memory.
        -> To build a conversational AI application, Flowise provides Conversation Chain.
        -> Conversation Chain internally manages:
            -> Conversation History
            -> Prompt Creation
            -> Chat Model Invocation

    -> Architecture

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
              Chat Model

    -> Components

        -> Buffer Memory
            -> Stores the conversation history.
            -> Retrieves previous conversations whenever requested.

        -> Conversation Chain
            -> Retrieves conversation history from Buffer Memory.
            -> Internally creates the final Prompt.
            -> Invokes the selected Chat Model.
            -> Stores the latest conversation back into Buffer Memory.

        -> Chat Model
            -> Communicates with the selected LLM.
            -> Generates the final response.

    -> Execution Flow

            User enters a question
                    ↓
            Buffer Memory retrieves previous conversation
                    ↓
            Conversation Chain internally combines:
                -> System Prompt
                -> Conversation History
                -> Current User Prompt
                    ↓
            Final Prompt is created
                    ↓
            Chat Model sends the request to the configured LLM
                    ↓
            LLM generates the response
                    ↓
            Conversation Chain returns the response
                    ↓
            Buffer Memory stores the latest conversation

    -> Current Chatflow

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
                OpenAI

    -> Difference from Task 1

        Task 1

            Chat Prompt Template
                    ↓
                LLM Chain
                  ↑
                  │
              Chat Model

            -> Stateless Chatflow.
            -> Every request is processed independently.
            -> Previous conversations are not remembered.

        Task 2

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
              Chat Model

            -> Stateful Chatflow.
            -> Previous conversations are remembered.
            -> Conversation context is automatically included in every request.

    -> Important

        -> Buffer Memory stores only the conversation history.
        -> Conversation Chain automatically retrieves conversation history from Buffer Memory.
        -> Conversation Chain internally creates the Chat Prompt Template.
        -> Unlike LLM Chain, Conversation Chain supports Memory.
        -> Flowise hides the prompt creation process inside Conversation Chain.
        -> In manual implementation (Spring AI / LangChain4j), we create the Prompt, retrieve Memory, and invoke the Chat Model explicitly.
