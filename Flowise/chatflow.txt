# Phase 1 : Conversation Chain (Stateful Chatbots)
################################################################################

====================================! ✅Task 1: Create our first Chatflow !=======================

    Objective:
        -> Build a basic AI application by connecting a Chat Prompt Template, Chat Model, and LLM Chain.
        -> Understand how Flowise visually represents the components used in manual AI application development.

    -> Definition

        -> Chatflow is a visual workflow used to build AI applications in Flowise.
        -> It is built on top of LangChain.
        -> Instead of writing code, we connect different LangChain Components (Nodes) visually.
        -> Each Node performs a specific responsibility, and together they form a complete AI application.

    -> Architecture

            User
                ↓
        Chat Prompt Template
                ↓
            LLM Chain
              ↑
              │
          Chat Model
                ↓
            LLM Response

    -> Components

        -> Chat Prompt Template
            -> Creates the final prompt by combining the System Message and Human Message.

        -> Chat Model
            -> Acts as the communication bridge between the application and the selected LLM.

        -> LLM Chain
            -> Receives the Prompt and Chat Model.
            -> Invokes the selected LLM.
            -> Returns the generated response.

    -> Execution Flow

            User enters a question
                    ↓
            Chat Prompt Template combines:
                -> System Message
                -> Human Message
                    ↓
            Final Prompt is created
                    ↓
            LLM Chain receives:
                -> Final Prompt
                -> Chat Model
                    ↓
            Chat Model sends the request to the configured LLM
                    ↓
            LLM generates the response
                    ↓
            LLM Chain returns the response
                    ↓
            Response is displayed to the user

    -> Current Chatflow

            Chat Prompt Template
                        ↓
                    LLM Chain
                      ↑
                      │
                Google Gemini

    -> Use Cases

        -> AI Chatbot
        -> Question Answering
        -> RAG Applications
        -> PDF Chat
        -> Knowledge Assistant
        -> Customer Support Bot

    -> Limitations

        -> Best suited for single-agent applications.
        -> Limited support for complex branching and multi-agent workflows.
        -> For advanced AI applications, Agentflow is recommended.

    -> Important

        -> Chatflow is built on top of LangChain.
        -> Every Node represents a LangChain Component.
        -> We build an AI application by connecting the required Nodes together.
        -> The Chat Prompt Template prepares the prompt.
        -> The Chat Model communicates with the selected LLM.
        -> The LLM Chain orchestrates the complete execution flow.

====================================! ✅Task 2: Add Memory to Chatflow !=======================

    Objective:

        -> Enable the Chatflow to remember previous conversations.
        -> Understand how conversation history is maintained and how it improves contextual responses.

    -> Why Conversation Chain?

        -> In Task 1, we used LLM Chain to execute a single prompt.
        -> LLM Chain is a stateless chain, meaning it does not remember previous conversations.
        -> Therefore, it cannot be directly used with Buffer Memory.
        -> To build a conversational AI application, Flowise provides Conversation Chain.
        -> Conversation Chain internally manages:
            -> Conversation History
            -> Prompt Creation
            -> Chat Model Invocation

    -> Architecture

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
              Chat Model

    -> Components

        -> Buffer Memory
            -> Stores the conversation history.
            -> Retrieves previous conversations whenever requested.

        -> Conversation Chain
            -> Retrieves conversation history from Buffer Memory.
            -> Internally creates the final Prompt.
            -> Invokes the selected Chat Model.
            -> Stores the latest conversation back into Buffer Memory.

        -> Chat Model
            -> Communicates with the selected LLM.
            -> Generates the final response.

    -> Execution Flow

            User enters a question
                    ↓
            Buffer Memory retrieves previous conversation
                    ↓
            Conversation Chain internally combines:
                -> System Prompt
                -> Conversation History
                -> Current User Prompt
                    ↓
            Final Prompt is created
                    ↓
            Chat Model sends the request to the configured LLM
                    ↓
            LLM generates the response
                    ↓
            Conversation Chain returns the response
                    ↓
            Buffer Memory stores the latest conversation

    -> Current Chatflow

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
                OpenAI

    -> Difference from Task 1

        Task 1

            Chat Prompt Template
                    ↓
                LLM Chain
                  ↑
                  │
              Chat Model

            -> Stateless Chatflow.
            -> Every request is processed independently.
            -> Previous conversations are not remembered.

        Task 2

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
              Chat Model

            -> Stateful Chatflow.
            -> Previous conversations are remembered.
            -> Conversation context is automatically included in every request.

    -> Important

        -> Buffer Memory stores only the conversation history.
        -> Conversation Chain automatically retrieves conversation history from Buffer Memory.
        -> Conversation Chain internally creates the Chat Prompt Template.
        -> Unlike LLM Chain, Conversation Chain supports Memory.
        -> Flowise hides the prompt creation process inside Conversation Chain.
        -> In manual implementation (Spring AI / LangChain4j), we create the Prompt, retrieve Memory, and invoke the Chat Model explicitly.
  
====================================! ✅Task 3: Prompt Templates !=======================

    Objective:

        -> Learn how to create dynamic prompts using variables.
        -> Build reusable prompts for different user inputs.

    -> Why Prompt Templates?

        -> In Task 1 and Task 2, we used static prompts.
        -> Static prompts require manual modification whenever the user input changes.
        -> This makes the prompt difficult to reuse.
        -> Prompt Templates solve this problem by introducing Prompt Variables.
        -> Prompt Variables allow the same prompt to be reused for different user inputs without modifying the prompt itself.

    -> Architecture

                    User
                      │
                      ▼
            Runtime Variable (question)
                      │
                      ▼
            Format Prompt Values
                topic → question
                      │
                      ▼
            Chat Prompt Template
                Explain {topic}
                      │
                      ▼
            Conversation Chain
                  ▲           ▲
                  │           │
            Chat Model   Buffer Memory
                      │
                      ▼
                 LLM Response

    -> Components

        -> Chat Prompt Template

            -> Creates a reusable prompt.
            -> Supports Prompt Variables using {}.

        -> Prompt Variables

            -> Placeholders used inside the prompt.
            -> Receive values before execution.

            Example:

                Explain {topic}

        -> Format Prompt Values

            -> Maps Prompt Variables to runtime variables or fixed values.

            Example:

                topic → question

        -> Runtime Variables

            -> Flowise provides runtime variables during execution.

            Available runtime variables:

                -> question

                    Current user input from the chat window.

                -> chat_history

                    Previous conversation between the user and AI.

                -> file_attachment

                    Files uploaded by the user.

        -> Conversation Chain

            -> Receives the final prompt after all variables are replaced.
            -> Sends the prompt to the Chat Model.

        -> Chat Model

            -> Sends the request to the configured LLM.
            -> Returns the generated response.

    -> Execution Flow

            User enters:

                Kafka

                    ↓
            Flowise creates runtime variable

                question = "Kafka"

                    ↓
            Format Prompt Values maps

                topic → question

                    ↓
            Chat Prompt Template replaces

                Explain {topic}

                    ↓

                Explain Kafka

                    ↓
            Final Prompt is created

                    ↓
            Conversation Chain receives the prompt

                    ↓
            Chat Model invokes the selected LLM

                    ↓
            LLM generates the response

                    ↓
            Response is displayed to the user

    -> Current Chatflow

                Chat Prompt Template
                        │
                        ▼
                Conversation Chain
                  ▲           ▲
                  │           │
              Chat Model   Buffer Memory

    -> Difference from Task 2

        Task 2

                Buffer Memory
                      │
                      ▼
            Conversation Chain
                  ▲
                  │
              Chat Model

            -> Focused on remembering previous conversations.
            -> Prompt was static.
            -> No Prompt Variables were used.

        Task 3

                User
                  │
                  ▼
            Runtime Variable
                  │
                  ▼
            Format Prompt Values
                  │
                  ▼
            Chat Prompt Template
                  │
                  ▼
            Conversation Chain
                  ▲           ▲
                  │           │
              Chat Model   Buffer Memory

            -> Prompt became dynamic.
            -> Prompt Variables were introduced.
            -> Same prompt can be reused for different user inputs.
            -> Variables are replaced before sending the prompt to the LLM.

    -> Important

        -> Prompt Variables are written inside {}.

            Example:

                {topic}
                {language}
                {difficulty}

        -> Every Prompt Variable must receive a value before execution.

        -> Prompt Variables can receive:

            -> Fixed values
            -> Runtime variables

        -> Runtime variables available in our Flowise version are:

            -> question
            -> chat_history
            -> file_attachment

        -> Example Mapping

            Prompt:

                Explain {topic}

            Mapping:

                topic → question

            User Input:

                Kafka

            Final Prompt:

                Explain Kafka

        -> If a Prompt Variable is not assigned, Flowise throws an error.

            Example:

                Missing value for input variable topic

        -> Variable replacement happens before the prompt is sent to the Conversation Chain.

        -> Prompt Templates work with both Conversation Chain and LLM Chain.

        -> Prompt Templates improve prompt reusability by separating the prompt structure from the runtime data.

====================================! ✅Task 4: Input Moderation !=======================

    Objective:

        -> Validate user input before sending it to the LLM.
        -> Prevent harmful or restricted prompts.

    -> Why Input Moderation?

        -> LLMs process every prompt they receive.
        -> Some prompts may contain harmful, confidential, or restricted content.
        -> Input Moderation acts as the first layer of validation before the prompt reaches the LLM.
        -> It improves security, prevents misuse, and reduces unnecessary LLM API calls.

    -> Types

        -> Simple Prompt Moderation

            Performs rule-based moderation using a configured Deny List.

        -> OpenAI Moderation

            Performs AI-powered moderation using OpenAI's Moderation API.

    -> Architecture

                    User
                      │
                      ▼
            Simple Prompt Moderation
                      │
                ┌─────┴─────┐
                │           │
            Match Found   No Match
                │           │
                ▼           ▼
          Return Error   Chat Prompt Template
                              │
                              ▼
                      Conversation Chain
                        ▲            ▲
                        │            │
                   Chat Model   Buffer Memory
                              │
                              ▼
                             LLM

    -> Components

        -> Simple Prompt Moderation

            -> Checks the user's prompt against a configured Deny List.
            -> Blocks the prompt if a match is found.
            -> Returns the configured Error Message.
            -> Does not invoke the LLM for blocked prompts.

        -> Deny List

            Contains restricted words or phrases.

            Example:

                hack
                passwords
                credit card
                kill

        -> Error Message

            Message returned when moderation blocks the prompt.

            Example:

                Your prompt contains restricted content.
                Please modify your request.

        -> Chat Prompt Template

            Receives the prompt only if moderation is successful.

        -> Conversation Chain

            Processes only validated prompts.

        -> Chat Model

            Sends the validated prompt to the configured LLM.

    -> Execution Flow

            User enters a prompt

                    ↓
            Simple Prompt Moderation

                    ↓
            Compare with Deny List

                    ↓

            Match Found?

                Yes
                    ↓
                Return Error Message

                No
                    ↓
                Continue Chatflow

                    ↓
            Chat Prompt Template

                    ↓
            Conversation Chain

                    ↓
            Chat Model

                    ↓
                 LLM Response

    -> Test Cases

        Test 1

            Deny List

                hack

            User Prompt

                How to hack a website?

            Result

                Blocked

        ------------------------------------------------

        Test 2

            Deny List

                passwords

            User Prompt

                Tell me about password.

            Result

                Allowed

            Reason

                "password" and "passwords" are different text.

        ------------------------------------------------

        Test 3

            Deny List

                passwords

            User Prompt

                Tell me about passwords.

            Result

                Blocked

        ------------------------------------------------

        Test 4

            Deny List

                passwords

            User Prompt

                Tell me about Passwords.

            Result

                Blocked

            Reason

                Matching is case-insensitive.

        ------------------------------------------------

        Test 5

            Deny List

                credit card

            User Prompt

                My credit-card was stolen.

            Result

                Allowed

            Reason

                The configured phrase was not found exactly due to the hyphen.

    -> Current Chatflow

            Simple Prompt Moderation
                        │
                        ▼
                Chat Prompt Template
                        │
                        ▼
                Conversation Chain
                  ▲            ▲
                  │            │
              Chat Model   Buffer Memory

    -> Difference from Task 3

        Task 3

            -> Focused on creating dynamic prompts.
            -> Introduced Prompt Variables.
            -> Prompt was always forwarded to the LLM.

        Task 4

            -> Focused on validating the user's input.
            -> Prompt may be blocked before reaching the LLM.
            -> Adds a security layer before prompt execution.

    -> Important

        -> Input Moderation executes before the prompt reaches the LLM.

        -> Simple Prompt Moderation performs rule-based text matching.

        -> Matching is case-insensitive.

        -> If a deny-list entry is found, the prompt is blocked immediately.

        -> The configured Error Message is returned to the user.

        -> Blocked prompts never reach the Chat Prompt Template, Conversation Chain, or LLM.

        -> Simple Prompt Moderation does not understand context or meaning.

        -> It is best suited for:

            -> Profanity filtering
            -> Company-specific keywords
            -> Confidential information
            -> Restricted topics
            -> Basic keyword filtering

        -> For semantic or AI-powered moderation, OpenAI Moderation should be used instead.
